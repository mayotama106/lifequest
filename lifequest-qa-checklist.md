# LifeQuest 手動QAチェックリスト (S6-2)

対象: `index.html`（HTML強化MVP）
自動テスト（DOM非依存ロジック）: `node lifequest-logic-tests.js` … **25項目パス済み**
本チェックリストは、自動化していない **DOM/UI 挙動** をブラウザで確認するためのもの。

## 永続化 (E1)
- [ ] クエストを完了 → リロード後も完了状態・LV・EXPバーが維持される
- [ ] BOSS討伐でスキル獲得 → リロード後もSKILLSに残る
- [ ] 融合実行 → リロード後も進化スキル・融合ボタン「統合済」が維持される
- [ ] RESET → 確認ダイアログが出る／キャンセルで何も起きない
- [ ] RESET実行 → 初期状態になり、リロードしても初期のまま
- [ ] localStorage を手動で壊す → エラーで落ちず初期状態で起動する

## クエストCRUD (E2)
- [ ] 「＋ ADD」でモーダルが開く（斜めパネル・黒地・白文字）
- [ ] クエスト名が空 → 登録できず枠が赤くなる
- [ ] 通常クエストを追加 → 一覧末尾に出る／残数カウント更新／リロード維持
- [ ] BOSSにチェック → スキル名欄が出る／空だと登録不可
- [ ] 習慣にチェックして登録 → 翌日（後述）にリセット対象になる
- [ ] 編集（✎）でモーダルに既存値がプリセットされ、更新が反映される
- [ ] 削除（🗑）で確認 → 一覧と残数から消える
- [ ] 編集/削除ボタンを押しても **完了トグルが発火しない**
- [ ] タスク名に `<script>` や `<b>` を入れても、タグとして解釈されず文字表示される（XSS対策 S6-1）

## 習慣・ストリーク (E3)
- [ ] 習慣クエスト完了 → ヘッダーの🔥数値が増える
- [ ] 端末日付を翌日に進めて再読込 → 習慣クエストが未完了に戻る／非習慣は維持
- [ ] 連続日で完了 → ストリークが増える
- [ ] 1日空けて再読込 → ストリークが0に戻る
  - ※日付テストは OS の日付変更、または DevTools で `lastSeenDate`/`lastCompletionDate` を localStorage 上書きして検証

## 成長・スキル・融合 (E4/E5)
- [ ] EXPが必要値に達するとレベルアップしトーストが出る（複数レベル同時も可）
- [ ] 3種の基礎スキルが揃うと融合ボタンが点滅・有効化（"スキル統合"）
- [ ] 融合演出（アナウンス→素材→閃光→新スキル）が再生される
- [ ] 融合完了 → 進化スキルがSKILLS先頭に、素材は "USED ▶ FUSED" 表示
- [ ] STATUS の称号が進化後に頂点称号へ変わる
- [ ] 「▶ スキル獲得 演出を見る」「▶ 進化 演出を見る」プレビューがゲーム状態を変えない

## 表示・アクセシビリティ
- [ ] STATUS タブの属性バー（実行力/討伐力/成長値）が完了状況に応じて伸びる
- [ ] OS の「視差効果を減らす（reduce motion）」ON で、カットイン/融合が瞬時に短縮される
- [ ] キーボード操作: クエストカードに Enter/Space で完了できる
- [ ] フォーカスリング（シアン）が表示される
